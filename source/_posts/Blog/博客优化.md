##### 性能优化

- SEO 优化（网站推广）
  - 让搜索引擎收录你的站点
    - 参考: <https://blog.csdn.net/u013732204/article/details/124176132>
      - 主动推送 > 自动推送 > 提交sitemap
      - [百度收录](https://ziyuan.baidu.com/)
      - [Bing收录](https://www.bing.com/webmasters/)
      - [google收录](https://search.google.com/search-console/)
  </br>

  - 优化description、keyword和title
    - 调查显示只有20%的网页用到“关键字”与“描述”标签（即keyword, description）
    - description
      - Next主题默认将摘要部分作为文章的description, 所以只需要添加keyword。
    - keywords
      - 首页推荐承载3-5个关键词
        - 每个关键词有唯一的主目标业务需求性页面并且与首页相互链接。
      - 内页推荐承载1-3个长尾关键词
      - 添加1-2个关键词到title中
      - 用[LSI](http://lsigraph.com)找相关关键词
    - 关于使用
      - 在文章Front-matter中添加description字段，多个用逗号隔开，并以逗号结尾。
      - 在`themes\next\layout\_partials\head\`下的head.swing添加如下代码

        ```yml
        {%- if page.keywords and page.keywords.length %}
          <meta name="keywords" content="{{ page.keywords }},{{ config.keywords }}" />
        {%- endif %}
        ```

  </br>

  - 优化站点的URL
    - 搜索引擎优化认为，网站的最佳结构是用户从首页点击三次就可以到达任何一个页面。
    - 修改站点的永久链接，如:`permalink: :category/:name/`
  </br>

  - 优化相关链接，外链自动添加nofollow
    - 做好站内各类页面之间的相关链接。
    - Google推荐使用nofollow，告诉搜索引擎爬虫无需抓取目标页，同时告诉搜索引擎无需将的当前页的Pagerank传递到目标页。（也不需要全部都加上，添加至少2个站外链接到相关的权威网站）
    - [hexo-filter-nofollow](https://github.com/hexojs/hexo-filter-nofollow)
  </br>

  - 丰富站点内容
    - 扩展站点内容，增强专业性
    - 增加部分原创内容
  </br>
  
  - 关于使用子域名还是子目录
    - 需要遵守的原则是：**宁缺勿滥**
      - 目录内容庞大  ->  子域名
      - 能够提供足够多的内容  ->  子域名
      - 搜索引擎的处罚机制  ->  滥用子域名
</br>

- 网站预加载JS脚本
  - instant.page:提高网站页面加载速度
    - PC端
      - 在用户单击链接之前，他们将鼠标悬停在该链接上。当用户悬停65ms时，有两个用户中的一个会单击该链接，因此instant.page此时开始预加载，平均留给页面预加载的时间超过300 ms。
      - 另一个选项是，当用户开始按下鼠标时，在释放鼠标之前进行预加载。这使得几乎没有未使用的请求，同时页面负载平均提高了80毫秒。
    </br>

    - 移动端
      - 用户在释放显示器之前开始触摸显示器，平均留出90毫秒供页面预加载。
      - 另一个选择是在链接可见时立即预加载它们。
    - Dependencies: <https://github.com/instantpage/instant.page>
  - 通过安装minify压缩JS文件
    - 安装minify: `npm i minify -g`
    - 压缩JS文件: `minify Instant.js > Instant.min.js`
</br>

- 自建CDN资源
  - CDN的全称是Content Delivery Network，即内容分发网络。
  </br>

  - CDN是构建在网络之上的内容分发网络，依靠部署在各地的边缘服务器，通过中心平台的负载均衡、内容分发、调度等功能模块，使用户就近获取所需内容，降低网络拥塞，提高用户访问响应速度和命中率。
  </br>

  - CDN的关键技术主要有内容存储和分发技术。
  </br>

  - 使用CDN加速来优化网站打开速度，jsDelivr + GitHub便是免费且好用的CDN。
  </br>

  - GitHub使用对应仓库的资源即可: <https://cdn.jsdelivr.net/gh/repository/jquery.min.js>
  - Dependencies: <https://www.jsdelivr.com/>
